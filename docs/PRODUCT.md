> 面向产品、运营、设计的使用说明与体验规范。技术实现见 [TECH_SPEC.md](./TECH_SPEC.md)，需求条目见 [PRD.md](./PRD.md)。

## 1. 产品是什么

AIMarket 是面向中文用户的 **AI 修图与电商设计平台**。用户用自然语言描述需求，系统理解意图、调用合适的图像模型，输出可下载的成品图。

**一句话**：修图不用学 PS，说话就能出图。

---

## 2. 核心价值主张

| 价值 | 说明 |
|------|------|
| 中文优先 | 无需拼凑英文 Prompt |
| 对话式迭代 | 不满意继续说，不用从头再来 |
| 电商开箱即用 | 套图 Agent 结构化收集卖点与平台规范 |
| 智能模型路由 | 按任务自动选模型（Phase 2 完整上线） |
| 透明计费 | 生成前可见积分预估 |

## 4. 页面说明

### 4.1 首页 `/`

**目标**：30 秒内让用户理解产品并尝试第一次输入。

| 区域 | 内容 | 设计要点 |
|------|------|----------|
| 顶栏 | Logo、登录、CTA | 渐变主按钮「免费开始」 |
| 公告条 | 活动信息 | 可关闭，不挡主操作 |
| Hero | 品牌语 + 吉祥物 | 深色 + 右上暖光 |
| 命令面板 | 三 Tab 输入 | 玻璃拟态大圆角容器 |

**主文案**：

- 标题：修图，就用 AIMarket  
- 副标题：让每个人都能轻松创作出专业级图片和视频  

### 4.2 工作台 `/studio`

**目标**：完成「上传 → 描述 → 生成 → 下载」闭环。

| 区域 | Phase 1 | Phase 2+ |

| 左侧 | 会话历史列表 | + 文件夹 |

## 5. 创作模式产品规则

### 5.1 对话模式

1. 用户输入文本，可选上传 1–4 张图。  
2. 选择模型、张数、分辨率档位。  
3. 提交后显示「生成中」状态。  
4. 完成后 assistant 消息展示图片，可下载。  
5. 继续输入即在同 session 上下文迭代。

**积分**：按 `estimatePoints` 预扣展示，任务成功后结算。

### 5.2 快速模式

- 单轮为主，默认「全能图片 V2」类高速模型。  
- UI 与对话共用面板，减少二次确认。

### 5.3 电商套图 Agent（Phase 2）

| 字段 | 必填 | 说明 |
|------|------|------|
| 产品图 | 是 | 实物/白底图 |
| 参考图 | 否 | 排版风格参考 |
| 品牌名 | 否 | 影响文案与 Logo 区 |
| 平台 | 是 | 淘宝/京东/抖音等 |
| 市场 | 是 | 影响文案语言习惯 |
| 语言 | 是 | 输出文案语言 |
| 产品详情 | 是 | 卖点、规格、受众、材质 |
| 设计师 | 否 | 风格人设 |
| 模型 | 是 | 默认推荐 Pro 档 |

## 7. 积分体系（产品规则）

| 项目 | 规则 |
|------|------|
| 货币名称 | 积分（内部代号 credits） |
| 新用户赠送 | 100 积分 |
| 扣费时机 | 任务进入 `running` 时预扣；`failed` 全额退回 |
| 预估因子 | 模型系数 × 张数 × 分辨率系数 |
| 不足提示 | 弹窗引导充值（Phase 3 支付） |

**Phase 1 计价表（示例）**：

| 模型档位 | 单张积分 |
|----------|----------|
| 标准（V2） | 10 |
| Pro | 20 |
| Fast 企业线 | 30 |

分辨率：1K ×1，2K ×1.5，4K ×2。
